# 개인 목표 문장 (v2)

> 이 대화 전체(OKF·하네스 비교·team-harness 재해석)가 뒷받침하는 정체. 입사 자기소개·목표설정·프로필에 쓸 무기.
> 정체성 단어 = **아키텍트** (비즈니스 플랫폼 채용공고의 '핵심 아키텍트' 역할과 직접 정렬).

## 한 문장 (정체)

> **나는 회사의 흩어진 데이터와 조직의 암묵지를 에이전트가 읽는 시스템으로 만들어, 비개발자가 AI를 안전하게 무기로 쓰게 하는 사내 플랫폼을 설계하는 아키텍트다.**

## 왜 나인가 (차별점 — 3줄)

1. **통제를 이미 마스터함.** team-harness에서 "사람이 git을 몰라도 엄격한 버전관리 + 순서·결과 보장이 자동으로 일어나는" 하네스를 만들어봄. → 자율/자동화를 *안전하게* 얹을 수 있는 드문 자산. (대부분은 자율만 알고 통제를 몰라 사고 냄.)
2. **에이전틱 개발 실전 경험.** Claude Code로 스킬·서브에이전트·게이트를 굴려봄 → 현업 통증을 빠르게 도구화하는 근육.
3. **목적지를 정확히 앎.** 필드 최전선(OKF)이 가리키는 "암묵지 → 기계가 읽는 데이터 스토리지 → 기업 AX"가 회사 비즈니스 플랫폼의 목적지와 동일. 동경이 아니라 지도를 이미 가짐.

## 짧은 형 (기억용)

> **"비개발자가 git을 몰라도 AI를 안전하게 쓰게 만드는 사람."**
